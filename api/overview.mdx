## Introduction

The Brautcloud Backend provides a RESTful API for managing wedding events, images, and user data. The API is built with Spring Boot and follows standard REST conventions with JSON request/response formats.

<Warning>
  **Authentication Currently Disabled**
  
  Spring Security is currently commented out in the project configuration. All endpoints are publicly accessible without authentication. This should be enabled before deploying to production.
</Warning>

## Base URL

The API is accessible at:

```
http://localhost:8080/api
```

All endpoints are prefixed with `/api` followed by the resource path.

The API is organized into three main resource categories:

## Authentication

<Note>
  Currently, no authentication is required to access the API endpoints. The `spring-boot-starter-security` dependency is commented out in `pom.xml`.
</Note>

Future implementation will include JWT-based authentication. For now, all endpoints are open and accessible without tokens or credentials.

## Common Response Formats

### Success Responses

Most GET endpoints return JSON arrays or objects:

```json
{
  "id": 1,
  "eventName": "Sarah & John's Wedding",
  "location": "Grand Hotel",
  "date": "2026-06-15T14:00:00",
  "userId": 1,
  "coupleName": "Sarah & John"
}
```

### Error Responses

The API uses standard HTTP status codes:

<ResponseField name="200" type="OK">
  Request succeeded
</ResponseField>

<ResponseField name="400" type="Bad Request">
  Invalid request parameters or missing required fields
</ResponseField>

<ResponseField name="404" type="Not Found">
  Requested resource does not exist
</ResponseField>

<ResponseField name="500" type="Internal Server Error">
  Server-side error occurred
</ResponseField>

## Error Handling

Errors are returned with appropriate HTTP status codes. Spring Boot's default error handling provides error details in the following format:

```json
{
  "timestamp": "2026-02-28T10:30:00.000+00:00",
  "status": 404,
  "error": "Not Found",
  "path": "/api/events/999"
}
```

## File Upload Limits

The API supports file uploads with the following constraints:

<ParamField path="max-file-size" type="string">
  Maximum file size: **10MB**
</ParamField>

<ParamField path="max-request-size" type="string">
  Maximum request size: **10MB**
</ParamField>

These limits are configured in `application.properties` and can be adjusted as needed.

## Content Types

### JSON Endpoints

Most endpoints accept and return `application/json`:

```bash
Content-Type: application/json
```

### File Upload Endpoint

The image upload endpoint uses `multipart/form-data`:

```bash
Content-Type: multipart/form-data
```
